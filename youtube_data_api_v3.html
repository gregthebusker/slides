Oauth 2 & You

OAUTH 2

  • Authentication and authorization are both handled via the OAuth 2 protocol for all modern Google APIs.
    • Unlike older GData APIs, OAuth 2 is the only supported means of auth.
  • Login and authorization happen in a web page hosted via HTTPS on google.com, to ensure that your credentials aren't shared with any third parties.
  • Access tokens expire and need to be periodically refreshed, which limits the exposure if an access token is compromised.
    • Google APIs client libraries handle token bookkeeping for you!
  • Different OAuth 2 flows depending on application type:
    • Client flow is used in JavaScript-only web applications that can't store refresh tokens.
    • Installed application flow is used with desktop/command line applications.
  • To support OAuth 2 in your app, you need to register in the API Access section of the Google API Console.

    scope: OAUTH2_SCOPES,
    immediate: true
  }, handleAuthResult);
}

YouTube Analytics API

YouTube Analytics API

  • The API consists of a single authenticated GET method that returns the result in a table like response (as JSON or CSV)
  • The HTTP parameters are similar to the Google Analytics API
  • Developers specify what data they want
  • Very flexible interface that allows you to specify many different queries
  • Only a limited number of queries are supported. You will need to work with the documentation to see what queries are supported
  • The API is not complete yet and we strive to make all our relevant reporting data available through the API
